COP By — Product Roadmap

Focus: usability and traction  ·  Revenue secondary for now

North Star

Make COPm / digital pesos the easiest way to spend from MiniPay for users in Colombia.

Primary metric

COP spent or sent per weekly active user (not just purchased).

MetricRole
Completed conversion (COPm purchase)Activation
Completed spend or send (transfer / BRE-B)Retention
7-day return rateTraction
Shares / referralsOrganic growth

Target user

AttributeDefinition
WhoMiniPay users in Colombia
InputThey receive USD (USDC, USDT, etc.) in their wallet
NeedConvert and spend in Colombian pesos
ProfileNot crypto-native; wants usable pesos, not a technical workflow
"Turn your MiniPay dollars into pesos you can actually use."

COPm is the onchain engine; users don't need to understand tokens.

Current state

CapabilityStatus

Buy COPm and receive it in another walletDone

Analytics (swaps + transfers)Done

Payment rails architecture

Two complementary paths, not mutually exclusive:

MiniPay wallet
    ├─ Buy COPm (Squid)       → pesos in wallet, P2P, Celo ecosystem
    └─ Pay via BRE-B (Abroad) → pesos in the real world (bank, Nequi, merchant)
RailProviderUse case
Onchain COPmSquid RouterMiniPay balance, P2P transfers
Fiat COP via BRE-BAbroad (under evaluation)Payments to accounts and merchants in Colombia

Distribution

ChannelTactic
MiniPayShort flow (<3 visible taps), amounts in COP, Colombian Spanish copy in the app
Colombian communitiesOne clear use case per activation ("pay in Colombia from MiniPay")
Word of mouthShareable receipt (WhatsApp), referrals, send to family
"Get paid in dollars on MiniPay? With COP By you convert them to pesos and send them to any account in Colombia."

Roadmap phases

Phase 0 Align product with the user ✅ Completed · Jun 2026

Overall goal: First-time visitors understand the value in seconds and have reasons to come back.

#TaskGeneral detailObjective

Success criteria

  • Purchase flow completion rate > 70%
  • Users returning within 7 days > 25%
Phase 1 First real spend: BRE-B with Abroad 4–8 weeks

Overall goal: Give users a concrete reason to return after converting USD — spend in the real Colombian economy.

Candidate provider: Abroad Finance — USDC/USDT → COP fiat via BRE-B.

#TaskGeneral detailObjective
1.1Outreach and API accessRequest API keys, sandbox, and workflow documentation from Abroad sales.Unblock integration
1.2Validate compliance requirementsConfirm KYC per user vs integrator, limits, minimums, and settlement times.Design UX without regulatory surprises
1.3MVP technical flowBackend: quote → payout → webhooks. Frontend: COP amount + BRE-B recipient + confirmation.First end-to-end BRE-B payment
1.4"Spend" tab in UIEvolve Buy | Transfer into Buy | Send | Spend.Unify conversion + spending in one app
1.5Persistence and analyticsLog BRE-B payouts in DB (similar to swaps/transfers).Measure adoption and debug failures
1.6Error handling and statesClear states: quoting, processing, completed, failed + messages in Spanish (app UI).Trust in fiat payments

Target flow

[How many pesos?] → [To whom? BRE-B key / account] → [Confirm] → [USDC from MiniPay] → [COP via BRE-B]

Open questions with Abroad (blockers)

  • MiniPay wallet integration via JWT (/walletAuth)?
  • KYC per end user or at integrator level?
  • Sandbox available before production?
  • USDC only or USDT as well?

Success criteria

  • > 15% of active users complete at least one BRE-B spend
  • > 1.2 spend transactions per weekly active user
